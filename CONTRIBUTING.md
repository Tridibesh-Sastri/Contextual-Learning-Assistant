# Contributing

Thanks for helping improve the project. Please follow the workflow below.

## Branching
- Create feature branches: `feat/<short-name>`
- Fix branches: `fix/<short-name>`

## Commit messages
Use a short prefix:
- `feat: <message>`
- `fix: <message>`
- `docs: <message>`
- `chore: <message>`

## Pull request checklist
- Explain what changed and why
- Update `CHANGELOG.md` for user-facing changes
- Bump `extension/manifest.json` version when releasing
- Add screenshots for UI changes if possible

## Code style
- Keep code readable and consistent with existing style
- Avoid heavy refactors inside feature PRs
